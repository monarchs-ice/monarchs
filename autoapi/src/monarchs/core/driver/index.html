src.monarchs.core.driver

Core functions used in the running of MONARCHS. This module contains the core functions that drive the code when it is executed.

When the code is run, it calls main(), which handles loading in the data, calling core.loop_over_grid every model day (interchangeably called “iteration”. The loop over each timestep (by default 1 hour) is handled in the single-column physics steps. main then calls the lateral movement functions, and handles saving the data, both in terms of the model state (also known as a “dump”), and the variables that the user wants to track over time.
